# 현재 진행 중인 플랜

> 시작: 2026-06-07
> 항목: 정적 가이드·블로그의 외부 폰트 로딩·LCP 등 페이지 성능이 검색 순위·체류에 충분한가

---

## 목표

외부 조건(트래픽·계정 신뢰도) 없이 **코드로 바로 효과를 보는 유일한 SEO 항목**. 정적 가이드·블로그 페이지는 현재 Google Fonts를 외부에서 로드(render-blocking)하고 있어 LCP·체류에 영향이 있을 수 있다. 추측이 아니라 **측정 기반**으로 실제 병목을 찾고, 비용 대비 효과가 큰 것부터 개선한다.

생각해야 할 문제:
- 측정 없이 추측 최적화는 금지 — 어디가 진짜 병목인가 (LCP/CLS/INP 중 무엇이 나쁜가)
- 폰트: self-host vs preconnect+font-display 트레이드오프 (유지보수·CDN 캐시 이점 vs render-blocking 제거)
- 검색 트래픽 대부분이 모바일 — 모바일 성능 우선
- 개선이 실제로 순위·체류로 이어지는지 어떻게 검증할 것인가 (착수 전/후 측정)
- 정적 HTML(가이드·블로그) vs React 앱(홈)은 성능 특성이 달라 따로 봐야 하는가

## 체크리스트
- [x] 현황 측정 — 홈·가이드·블로그 대표 페이지의 LCP·CLS·INP 실측 (로컬 Lighthouse, 모바일)
- [x] 외부 폰트 로딩 점검 — 실측 결과 가이드·블로그는 `display=swap`으로 이미 우수, 폰트는 병목 아님
- [x] 이미지 최적화 점검 — 블로그 이미지 dimensions 누락 확인, 단 현 CLS 0이라 비병목(향후 보험)
- [x] render-blocking CSS/JS·기타 리소스 점검 — 홈 unused JS 156KB는 AdSense(수익), 압축은 이미 brotli
- [x] 비용 대비 효과 큰 항목부터 개선 적용 — 무료 범위 2건 적용·검증 완료 (아래)
- [ ] 개선 후 재측정 — 배포 후 라이브 LCP·CLS 재측정 + N일 뒤 순위·체류 (시간 지연, 후속 분리)

## 적용한 개선 (2026-06-07, 무료 범위)
**① 홈 CLS 0.431 → 0.0000 (검증 완료)**
- 원인: `<div id="root">` SEO fallback의 헤더·히어로가 React 앱과 박스모델 불일치(특히 히어로 상단 패딩 56px 차) → 마운트 교체 시 점프.
- 수정: `index.html` fallback 헤더·main·히어로를 App.jsx와 **동일 클래스 마크업**(`.wrap`/`.header-inner`/`.logo`/`.hero`)으로 재작성, 히어로 텍스트도 i18n `heroDesc`와 일치. 프로덕션은 `<head>`에 CSS `<link>`가 있어 fallback 페인트 시점에 클래스 적용됨.
- 검증(로컬): 프로덕션 빌드+Lighthouse → CLS `0.431→0.0000`. 헤드리스 DOM 덤프로 React 실제 마운트(`<select class=lang-select>` 등장, fallback span 소멸) 확인 → 거짓 양성 아님. SEO 본문 텍스트 전부 보존.
- 검증(라이브, 배포 후): 3회 측정 중 워밍 2회 일관 **CLS 0.0123, Perf 84~91, LCP 2.9~3.9s**. 1회차(0.17/Perf64)는 배포 직후 콜드 머신 단발 노이즈. → Core Web Vitals CLS **통과**(<0.1) 확정.

**② 콜드스타트 완화 — `fly.toml` `auto_stop_machines: 'stop'→'suspend'`**
- 첫 방문 풀부팅(curl 실측 3.5s) → 메모리 스냅샷 재개(~1s). `min_machines_running=0` 유지 = 무료.

## 실측 결과 (2026-06-07, 모바일 Lighthouse)
| 페이지 | Perf | LCP | CLS | TBT | 판정 |
|--------|------|-----|-----|-----|------|
| 홈 (React) | **57** | 4.1s | **0.431** | 110ms | 나쁨 |
| 가이드 (정적) | 94 | 2.4s | 0.005 | 0 | 우수 |
| 블로그 (정적) | 92 | 2.6s | 0 | 560ms | 우수 |

**핵심 반전**: 플랜이 의심한 "정적 가이드·블로그 외부 폰트 → LCP 악화"는 거짓. SEO 대상 페이지는 이미 92~94점·CLS≈0. 진짜 병목은 둘 뿐:
1. **홈 CLS 0.431** — `<div id="root">`의 SEO 크롤러용 정적 fallback(~290줄)을 `createRoot().render()`가 마운트 시 통째 교체(hydrate 아님) → 큰 레이아웃 점프. (※ AdSense는 미등록이라 무관 — 초기 오판 정정)
2. **콜드스타트 TTFB 3.5s** — 첫 방문(머신 잠듦). 워밍 후 0.7~1.2s. `fly.toml min_machines_running=0`.

둘 다 폰트/이미지/CSS 문제가 아니며 **트레이드오프(비용·광고수익·SEO 아키텍처) 결정이 필요** → 무단 변경 금지.
