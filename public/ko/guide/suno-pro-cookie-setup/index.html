Suno Pro 쿠키 설정 — 무손실 WAV 다운로드 켜기

SunoDown에 Suno Pro 세션을 연동하는 실무 가이드. 비밀번호는 일절 사용하지 않고 세션 쿠키 하나만 사용합니다. 메인 페이지의 모달보다 깊게 들어가 "왜 안전한지·만료되면 어떻게 되는지"까지 답합니다.

업데이트: 2026-05-26 · 읽는 데 약 3분 · 필요한 것: 같은 브라우저에서 활성화된 Suno Pro/Premier 구독

시작하기 전에

이 가이드는 WAV를 받고 싶을 때만 필요합니다. MP3 다운로드(ZIP 일괄 포함)는 쿠키 없이 무료입니다 — 기본 절차는 다운로드 방법 가이드를 참고하세요. 어느 형식을 받아야 할지 헷갈린다면 MP3 vs WAV 비교 한 페이지에 정리되어 있습니다.

5단계 설정

1단계

suno.com에 로그인

브라우저에서 suno.com에 접속해 Pro 또는 Premier 구독이 있는 계정으로 로그인합니다. 인증되어야만 __client 쿠키가 생성됩니다 — 로그인 전이라면 3단계에서 해당 행이 보이지 않습니다.

2단계

개발자 도구 → Cookies

F12(macOS는 Cmd+Option+I)로 개발자 도구를 엽니다. 쿠키 보기는 브라우저별로 탭 위치가 다릅니다:

Chrome / Edge

Application 탭 → Storage → Cookies → https://suno.com

Firefox

Storage 탭 → Cookies → https://suno.com

Safari

설정 → 고급에서 개발자 메뉴 활성화 → Storage 탭 → Cookies → https://suno.com

Brave / Arc / Opera

Chrome과 동일(Chromium 기반은 모두 동일한 DevTools를 공유합니다).

3단계

__client 값 복사

이름이 정확히 __client인 행을 찾아 클릭한 뒤, Value 필드 전체를 복사합니다. 보통 eyJ로 시작하는 긴 문자열이며 끝까지 모두 복사해야 합니다. 이름·도메인·다른 필드는 복사하지 마세요.

4단계

SunoDown에 붙여넣기

suno-down.com의 헤더 Pro 배지("WAV용 Pro 연결")를 클릭합니다. 복사한 값을 입력란에 붙여넣고 연결을 누르세요. SunoDown이 Suno에 세션을 검증하고 플랜 등급을 확인합니다 — 1~2초 내에 녹색 "Pro 연결됨" 배지가 표시됩니다.

5단계

형식 토글을 WAV로 전환

플레이리스트를 불러온 뒤 형식 토글을 MP3에서 WAV로 바꾸고 다운로드합니다. 각 WAV는 Suno 변환 작업을 트리거한 뒤 브라우저로 스트리밍되며, 곡당 몇 초가 걸립니다. ZIP 일괄 다운로드도 동일하게 작동합니다.

SunoDown이 쿠키로 실제로 하는 일

보안 모델 — 한눈에

  • 비밀번호는 사용하지 않습니다. __client 쿠키는 Suno의 자체 로그인 절차가 브라우저에 심어둔 세션 토큰입니다. SunoDown은 이메일·비밀번호를 절대 보지 않습니다.
  • 서버 메모리에만 보관됩니다. 쿠키는 SunoDown 서버 프로세스의 Map에 들어 있으며, 데이터베이스·로그·디스크에 기록되지 않습니다. 서버 재시작 시 모든 연결 세션이 사라집니다.
  • 연결 해제 즉시 폐기됩니다. Pro 모달의 "연결 해제" 클릭 시 즉시 삭제, Suno 측 세션 만료·명시적 로그아웃 시에도 동일합니다.
  • 단 하나의 용도로만 사용됩니다. SunoDown은 사용자 명의로 Suno의 WAV 변환 엔드포인트를 호출하고 약 50초마다 단기 JWT를 갱신합니다. 그 외에는 사용자 Suno 계정을 둘러보거나 변경하지 않습니다.
  • 구독이 이미 부여한 권한만 받습니다. 쿠키가 SunoDown에 사용자 구독 범위 이상의 권한을 줄 수 없습니다. 크레딧 구매·플랜 변경은 불가합니다.

전체 처리 정책은 개인정보 처리 방침에서 확인할 수 있습니다.

세션 만료와 재연결

Suno 인증은 장기 리프레시 쿠키(__client)와 거기서 파생된 단기 JWT로 구성됩니다. SunoDown은 JWT 갱신을 약 50초마다 자동 처리하므로, 한 번 연결하면 긴 다운로드 세션 동안 유지됩니다. 쿠키 자체가 무효화되는 경우는 다음과 같습니다:

  • 같은 브라우저에서 suno.com에서 로그아웃
  • 비밀번호 변경·기기 검토 등으로 Suno가 세션을 회전
  • Pro/Premier 플랜 만료 — 쿠키는 남지만 WAV 엔드포인트가 거부

이때 SunoDown은 "세션 만료" 배너를 띄웁니다. 재연결 절차는 동일합니다 — 2~4단계를 새 __client 값으로 반복하면 됩니다.

트러블슈팅

붙여넣자마자 "연결 실패"
거의 항상 값이 잘렸거나 불필요한 문자가 섞인 경우입니다. DevTools에서 Value 필드 전체를 다시 복사하세요 — 이름·따옴표는 빼고, 긴 문자열은 끝까지 포함해야 합니다. 수백 자가 정상입니다.
연결은 성공했는데 WAV 다운로드가 401
쿠키는 유효하지만 JWT가 중간에 어긋난 경우입니다. 연결 해제 후 다시 연결하면 깨끗하게 갱신됩니다. 반복되면 suno.com에 다시 로그인해 새 쿠키 값을 복사하세요.
연결 성공 후에도 "WAV는 Suno Pro 플랜 필요" 메시지
결제 등급이 Pro로 인식되지 않은 경우입니다. suno.com에서 구독이 활성 상태이고 정확한 계정으로 로그인되어 있는지 확인한 후 쿠키를 다시 복사하세요.
몇 분 사용 후 연결이 끊김
JWT 갱신이 실패하고 복구도 되지 않은 경우입니다. 수동으로 재연결하세요 — 전체 절차가 1분 이내입니다. 같은 브라우저에서 반복적으로 끊긴다면 Suno가 세션을 회전했을 가능성이 높으니 suno.com에 다시 로그인 후 진행하세요.
__client 행이 아예 안 보입니다
해당 브라우저에서 suno.com에 로그인되어 있지 않은 경우입니다. 새 탭에서 suno.com을 열고 로그인한 뒤 DevTools를 새로고침하면 즉시 표시됩니다.

자주 묻는 질문

Suno 쿠키를 SunoDown에 붙여넣어도 안전한가요?
쿠키는 서버 메모리에만 보관되며 데이터베이스나 로그 파일에 기록되지 않습니다. '연결 해제' 클릭, 세션 만료, 서버 재시작 시 즉시 폐기됩니다. 쿠키에는 Suno 비밀번호가 포함되지 않으며, Suno가 사용자 브라우저에 발급한 세션 토큰일 뿐입니다.
왜 이메일·비밀번호로 로그인하지 않나요?
Suno는 Clerk 인증을 사용하며 CAPTCHA·봇 차단으로 보호됩니다. 제3자 도구가 이를 우회할 수 없기 때문에, 사용자가 이미 인증한 세션의 쿠키를 받아 비밀번호를 절대 보지 않고 본인 명의로만 동작합니다.
쿠키는 얼마나 오래 유효한가요?
Suno의 __client 쿠키는 장기 리프레시 토큰이며, 거기서 파생되는 단기 JWT는 SunoDown이 약 50초마다 자동 갱신합니다. suno.com에서 로그아웃하거나 Suno 측에서 세션을 회전하면 새 값을 다시 복사해야 합니다.
MP3만 받는데도 Pro가 필요한가요?
필요 없습니다. SunoDown의 MP3 다운로드는 계정 없이 완전 무료입니다. Pro 쿠키는 유료 구독자에게만 열려 있는 무손실 WAV 엔드포인트를 열기 위한 것입니다.
쿠키를 붙여넣었는데도 다운로드가 401로 실패합니다.
401은 보통 쿠키가 더 이상 유효하지 않다는 뜻입니다. 같은 브라우저에서 suno.com에 접속해 로그인 상태를 확인한 후 2~4단계를 다시 진행하세요. 계속 실패하면 Pro 결제 상태가 만료된 것일 수 있으니 suno.com에서 결제 상태부터 확인해 주세요.
SunoDown이 제 Suno 계정에 결제를 일으킬 수 있나요?
그럴 수 없습니다. SunoDown은 사용자의 기존 Pro 구독이 이미 부여한 권한 안에서만 동작하며, 크레딧 구매·플랜 업그레이드·결제 변경을 일으킬 수 없습니다. 사용자가 이미 재생 가능한 트랙에 대해 WAV 변환 엔드포인트만 호출합니다.
